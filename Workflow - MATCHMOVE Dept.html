Workflow - MATCHMOVE Dept

    INPUTS:

  • MATCHMOVE gets data from Editorial Dept as depending on clients.
  • MATCHMOVE gets .JPEG, .obj, .ma, .fbx & .exr file format from editorial Dept.
  • MATCHMOVE uses camera info like focal length, film-back , focus distance, camera height and lidar scan etc, which is provided by client.
  • MATCHMOVE uses char-rig for body track.
  • MATCHMOVE uses 3DEqualizer, Maya Nuke as their working environment.

    OUTPUTS:

  • MATCHMOVE sends data to ANIM, FX,LIGHTING and COMP.
  • MATCHMOVE sends data to down-strem dept's in .ma , .abc , .fbx, .nk formats.

    ISSUES

    While publishing

    1. camera should be able to publish .ma , .abc , .fbx formats.
    2. MatchMove-Geo should be able to publish .ma , .abc , .fbx formats.
    3. Objects or Characters cache must be able to publish only in .abc.

    Dailies for

    1. WireFrame playblast
    2. Cone playblast
    3. Shaded playblast
    4. Outline playblast
    5. Perspective playblast
    6. Point blast playblast
