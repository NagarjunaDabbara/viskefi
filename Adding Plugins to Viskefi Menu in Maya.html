Adding Plugins to Viskefi Menu in Maya

This guide outlines the steps to integrate a developed or third-party plugin into the Viskefi space within Maya's ShotGrid integration. Follow these instructions to ensure the plugin is added successfully and deployed across the studio environment

A. Overview

This process includes:

  • Creating a Jira ticket for tracking.
  • Identifying the correct placement for the plugin in the Viskefi menu.
  • Adding and organizing the plugin source code in the Maya repository.
  • Testing and deploying the plugin for studio-wide availability.

    • B. Steps to Add a Plugin

  • Step 1: Create a Jira Ticket and Branch
  • After developing or acquiring the plugin, log the event by creating a Jira ticket to track progress.
  • Create a dedicated branch in the Maya repository corresponding to the Jira ticket.
    1. Follow branch-naming conventions for consistency and traceability.

    Step 2: Determine Plugin Placement

  • Identify the appropriate section in the Viskefi menu for the plugin.
    1. For example, if the plugin belongs to the General tab, plan its placement within this section.
  • The placement should align with the existing tab hierarchy in the Viskefi space.

    • Step 3: Add the Plugin Source Code

    1. Open the Maya repository and navigate to the directory where Viskefi menu space is defined.
      1. Locate the corresponding section or tab under which the plugin will reside.
    2. For example, if you are adding a plugin named RandomSelect under the General tab, the structure might look like this:-

      3. Example Path: maya/viskefi_menu/scripts/menu/general/RandomSelect.py

    3. Place the plugin's source code files into a new directory, adhering to the structure:

      4. Example Path:- maya/third-party/scripts/AdvancedSkeleton.mel

    Step 4: Self-Test the Plugin

    1. After placing the source code:
      1. Run local tests to confirm the plugin integrates with the Viskefi menu.
      2. Build the branch in viskefi-maya Repo by REZ build -i -v
      3. Get into the REZ env viskefi_maya environment.
      4. Launch maya from SG
      5. Verify the plugin functionality and UI compatibility.
    2. Document any dependencies or required configurations.

    Step 5: Commit and Push

    1. Commit the changes to the branch created in Step 1.
    2. Push the branch to the repository and inform the internal testing team.

    Step 6: Internal Testing

    1. Request internal testers to validate the plugin.
      1. Ensure functionality and UI performance are aligned with the studio's standards.
      2. Check for compatibility with existing plugins and menus.
      3. Resolve any issues or bugs reported during testing.
    2. Resolve any issues or bugs reported during testing.

    Step 7: Deployment

    1. Once testing is complete and the plugin works as expected:
      1. Merge the branch into the main repository following the studio's deployment guidelines.
      2. Deploy the plugin to the studio environment.
    2. Update documentation to include the new plugin and notify users.
