Workflow - Animation Dept

Inputs

  • Rig Wrappers: Updates of rigs being done manually.
  • Camera:
    • Manual references being done.
    • Full QC: Right now being done by the artist itself.
  • Textures:
    • Done manually after requesting Assets.
    • Manual texture assignment for the work.
  • Environment:
    • Artist will share the path.
    • Manual referencing of the files.

Outputs

  • Playblast:
    • Write in temp.
    • Updated to upload via browser.
  • Maps Folder: Shot grid for final EXR/MP and staging for the library.
  • Automatic Cache:
    • Exported manually to a server location and share the path.
    • Each character is exported separately.
    • Cameras are exported separately.
    • Environment is also baked separately from Animations to Lighting.

Issues

  • Assets are still able to see rig work files from version 1 even if it is the latest version.
  • Rigs have invisible functions once loaded after rig use.
  • Unknown QC from guides to other locations.
  • Animators are not getting the solved caches.
  • Assets do partial revisions of all the modifications in the studio.
  • Not every rig is following the naming conventions.
  • Final playblast still has to be exported manually from a shared location in the studio.
  • Ambient occlusion lighting is not added to all models.
  • Shots for short-term review need written or keepable tools.
  • Ambient animation lighting needs to be distributed across individual progress bars instead of a single deliverable.
  • Would be good to publish playblast to changed details from maps.
  • Errors on how to upload playblast from maps to staging.
